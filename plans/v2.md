# e-mona v2 Roadmap

Features and improvements for v2 — after v1 is shipped, stable, and used by real people.

**Rule:** Only start v2 features after v1 is published on Google Play and used for at least 1 month. Real usage reveals what matters most.

---

## High Priority (Most Requested / Most Impact)

### Custom Budget Period
Allow users to set a custom date range (e.g., 25th to 24th) instead of calendar month only. Requires rethinking month generation, purchase grouping, and analytics date boundaries.

### Recurring Purchases
"Every Monday I buy coffee for €3.50" — auto-suggest or auto-create recurring purchases. Stored as templates with frequency (daily/weekly/monthly) and auto-prompted when due. Reduces daily entry effort.

### Shared Shopping List
Real-time shared list: "We need milk, bread, eggs." Family members add items, one person checks them off while shopping, then converts the list to a purchase with one tap.

### Receipt Photo Scanning (OCR)
Take a photo of a receipt → auto-extract items, prices, and store name. Uses Google Cloud Vision API or on-device ML. Biggest time-saver for heavy users.

### Localization (i18n)
Translate the app to Bulgarian, German, Spanish, etc. Vue-i18n integration. v1 is English-only. Bulgarian is the first priority (home market).

---

## Medium Priority (Nice to Have)

### Multi-Currency per Budget
A single budget can have purchases in different currencies (e.g., EUR + USD while traveling). Requires exchange rate API (fixer.io or exchangerate.host), currency field per purchase, and conversion to the budget's base currency for totals.

### Budget Templates
"Student Budget", "Family Budget", "Freelancer Budget" — pre-configured templates with typical fixed costs, categories, and goals. New users pick a template instead of building from scratch.

### Subcategories
Allow nested categories (Food → Groceries, Restaurants, Snacks). v1 has flat categories only. Requires UI tree picker and analytics drill-down by subcategory.

### Smart Insights & Predictions
AI-powered analysis: "You spent 30% more on eating out this month", "At this rate you'll overspend by €120", "You always spend more on weekends." Can be a Cloud Function that runs weekly or client-side pattern detection.

### Bill Splitting
Split a purchase among budget members: "Dinner was €80, split 50/50 with Maria." Tracks who owes what within the shared budget. Shows a settlement summary.

### Budget Goals with Deadlines
"Save €3,000 for a car by December 2027" — progress tracking with deadline countdown, required monthly amount, and on-track/off-track indicator. Extends v1's yearly goals.

### Budget Comparison
Compare two months side-by-side: "March vs February — you spent €120 less on eating out but €80 more on transport." Visual diff with green/red indicators.

### Data Import
Import from bank CSV/OFX files to backfill historical spending data. Match imported transactions to categories using keyword rules (configurable per user).

---

## Lower Priority (Polish & Extras)

### Home Screen Widgets
Android widget and iOS widget showing remaining balance + quick-add button. Uses Capacitor community widget plugin or native module.

### Multi-Budget Switching
Quick switch between budgets (e.g., "Family" and "Personal side budget") without logging out. Already partially supported by `activeBudgetId` in v1's user schema — needs UI picker.

### Purchase Tags
Cross-cutting labels beyond categories: "vacation", "birthday", "work expense". A purchase can have multiple tags. Enables custom filtered views and tag-based analytics.

### Gamification
Achievements and badges: "7-day logging streak", "Under budget 3 months in a row", "First €1,000 saved." Motivates consistent daily usage.

### Web Dashboard
Full web version (same Vue.js codebase, no Ionic) for desktop access. Review analytics on a big screen, manage categories and settings more comfortably. Shares Firebase backend.

### Biometric Lock
Fingerprint / Face ID to open the app. Capacitor biometric plugin. Extra privacy for financial data on shared devices.

### Export to Google Sheets
Direct integration: one tap → create or update a Google Sheet with the current month's data. More useful than CSV for users who want ongoing spreadsheet tracking.

### Dark Mode (Full)
v1 may have basic dark mode support (Phase 7 bonus). v2 would be a fully polished dark theme with tested contrast ratios, chart colors adjusted for dark backgrounds, and a manual toggle (not just system preference).

---

## Decision Framework

When choosing which v2 features to build first, ask:
1. **How many users asked for it?** (Check app store reviews, feedback channel)
2. **Does it reduce daily friction?** (Recurring purchases, OCR, shopping list)
3. **Does it bring new users?** (Localization, templates, web dashboard)
4. **How complex is it?** (Start with quick wins, defer multi-month efforts)
